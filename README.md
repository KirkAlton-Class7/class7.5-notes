# Class7-5 / SEIR-1 Notes 

## Why
If I am teaching, I will typically use these notes to organize class structure, provide documentation, static assets, artifacts, scripts, etc. 

Sometimes I will take notes if I am not teaching for demonstration purposes: to prepare for other classes, if it’s new material, or for other similar reasons. 

## Usage
Treat my notes as an *additional* resource. Ideally, you aren't looking at them during class but are instead writing your own. 

You should be taking your own notes. This is a skill you need to develop not only to pass certifications or study but to bolster your technical writing skills. You need these documentation authoring skills as a critical "soft skill." On the job, you will be expected to:
* **Document issues:** Efficiently and concisely document issues you encounter, expected behavior, observed behavior, and possible solutions (e.g., to write tickets).
* **Summarize resources:** Summarize other documentation and resources you have used so that team members can quickly acquire the same knowledge (e.g., if you are tasked with researching a tool or testing a solution).
* **Create Runbooks:** Write out a method and strategy for lower-level engineers to solve problems independently.

## Organization
I will organize notes into weeks and, if needed, days. A week or day folder may contain:
* **README.md**: A Markdown file that serves as the core of the notes.
* **Supplemental Docs**: Additional Markdown files.
* **Assets Folder**: Contains diagrams and images.
* **Code/scripts Folder**: Contains shell scripts, Python, or Terraform files.

## Disclaimer 

I can only guarantee these notes accurately reflect class if I teach the class. If another teacher is giving instruction, they might cite different documentation, use different diagrams, or write code slightly differently. 

My notes should always be up-to-date and work correctly while arriving at the same solution, but how I explain, automate, or architect a solution could vary. In general, it should be essentially the same. Any differences would be a great opportunity for you to ask why I did it one way as opposed to someone else. 

Finally, **Theo has the final say**. If he wants something done a certain way, then that should always be presumed to be the way all instructors or admins expect it to be done.
